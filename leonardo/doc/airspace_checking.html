Leonardo "Airspace checking"

1.  Import of "openair files"

The admin must put the openair file (for example "Air_Europe 2006.txt") in the leonardo/data/airspace directory.
(and creat this dir if it doesnt exist).

Then from the top menu -> "Admin"-> "Administer Airspace checking" and

1) " Delete all airspace entries in the DB " to be sure no previous data exist, if it is the first run or you want to start all over.

2) Click on the "Update / Import airspace into DB" next to the airspace file. This may take from 5 to 20 secs.

 

NOTE: you can "Update / Import airspace into DB" the same file over and over again, if the airspace names are the same in the new/updated file
the comments you have inserted in the DB will be preserved.

 

2. Use

Airspace checking is automatic on flight submission. No message is displayed to the end user.

On the flight list, flights with airspace violations have the last column with a differfent background.

Also the initials of the "checker" are displayed if any.

Flights that havent been checked on submission (because they were submitted before this functionallity became available) are checke when ever the "edit flight" or the "show flight" screen are displayed. In the "edit flight" more info is displayed about the "check" and the admin has the option of manually making the check "pass" or "fail"

Also by setting the option to "Not yet proccessed" and pressing "submit changes" he will force the re-checking of the flight on the next time he enters this screen.

He can also enter his initials or a comment in the relevant field. He can enter for example "SJ Premission for airspace was granted". In the flight list only the first part till the first space will be displayed ie "SJ"

The vertical height inside the airspace area is always computed as the minimum one between "base" and "top". For example a flight inside an airspace of base=1000m and top=2000m with a gps point having alitude1100 will produce a 100m vertical violation. A point of 1900m will also produce a 100m violation

 

3. Disabling an airspace area.

In the "update Comments" subwindow there is a "disable area" checkbox.
You just check it and press "update". The now "disabled" area will still be listed in the flight
(marked as disabled but still invalidating the flight) until a new recheck is forced.
When the re-check is forced the "disabled" area will disappear from the list.

If this area was violated with other flights also, it will remain and invalidate these flights till a re-check is done for them.

There is no option to delete an area from within the interface, if there REALLY a need to do so instead of "disable" it
the admin can delete the DB record directly. It is however adviced to use the "disable" functinality because it persists
(together with the "Comments" field) through new airspace file update. This means that if the new airspace file is imported
the next year, areas that were deleted will re-enter the DB where as "disabled" areas will remain disabled (if they still exist in the new file).

All disabled areas are listed in the "Adminster Airspace checking" so the admin can edit their comments or re-activate them.
By re-activating an area all flights that violated this area must be manually be re-checked.
